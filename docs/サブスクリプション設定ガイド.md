## 1. 商品IDの確認

アプリに実装済みの商品IDは以下の2つです。  
ストアに登録するときは **必ずこのIDと一致させてください**。

| プラン | 商品ID | 価格 |
|--------|--------|------|
| 月額プラン | `premium_monthly_2500` | ¥2,500/月 |
| 年額プラン | `premium_annual_25400` | ¥25,500/年（約15%割引） |

## 2. App Store Connect での設定

### 2-1. サブスクリプショングループを作成する

1. [App Store Connect](https://appstoreconnect.apple.com) にログイン
2. 対象アプリ → **「App内課金」** → **「管理」**
3. **「＋」** → **「自動更新サブスクリプション」** を選択
4. **サブスクリプショングループ名** を入力（例：`プレミアムプラン`）  
   ※ グループ名はユーザーには表示されません

### 2-2. 月額プランを登録する

| 項目 | 入力値 |
|------|--------|
| 参照名 | `Premium Monthly 2500` |
| 製品ID | `premium_monthly_2500` ← アプリのコードと一致させる |
| サブスクリプション期間 | **1か月** |
| 価格 | **¥2,500**（価格レベル25相当） |

**ローカリゼーション（日本語）の設定**

| 項目 | 入力値 |
|------|--------|
| 表示名 | `プレミアムプラン（月額）` |
| 説明 | `AIが食事・トレーニングをフルサポート。写真解析、栄養アドバイス、トレーニングプラン生成など全AI機能が使えます。` |

### 2-3. 年額プランを登録する

| 項目 | 入力値 |
|------|--------|
| 参照名 | `Premium Annual 25400` |
| 製品ID | `premium_annual_25400` ← アプリのコードと一致させる |
| サブスクリプション期間 | **1年** |
| 価格 | **¥25,500**（月額より約¥4,500お得） |

**ローカリゼーション（日本語）の設定**

| 項目 | 入力値 |
|------|--------|
| 表示名 | `プレミアムプラン（年額）` |
| 説明 | `月額より15%お得な年額プラン。全AI機能が1年間使えます。` |

### 2-4. 無料トライアルを設定する（任意・推奨）

各プランの **「オファー」** タブ → **「イントロダクトリーオファーを追加」**

| 項目 | 推奨設定 |
|------|----------|
| オファータイプ | **無料** |
| 期間 | **7日間** または **14日間** |
| 適用条件 | 新規サブスクリプション初回のみ |

### 2-5. 審査情報を入力する

Appleの審査チームが確認するための情報が必要です。

| 項目 | 入力内容 |
|------|----------|
| レビューのスクリーンショット | サブスク画面のスクショ |
| レビューノート | 「AI機能（写真解析・栄養アドバイス・トレーニングプラン生成）が使えます。サブスク未加入の場合は自前のAPIキーでも利用可能です。」 |

### 2-6. App Store での価格改定について

- Apple の価格レベル表から近いものを選びます（¥2,500 はレベル25）
- 為替変動で自動調整を有効にしておくと便利です
- 価格変更は翌月から反映（既存ユーザーへ通知が送られます）

## 3. Google Play Console での設定

### 3-1. サブスクリプションを作成する

1. [Google Play Console](https://play.google.com/console) にログイン
2. 対象アプリ → **「収益化」** → **「定期購入」**
3. **「定期購入を作成」** をクリック

### 3-2. 月額プランを登録する

| 項目 | 入力値 |
|------|--------|
| 製品ID | `premium_monthly_2500` |
| 名前 | `プレミアムプラン（月額）` |
| 説明 | `AIが食事・トレーニングをフルサポートします。` |

**基本プランを追加**

| 項目 | 入力値 |
|------|--------|
| 基本プランID | `monthly-base` |
| 有効期間 | **1か月** |
| 自動更新 | **有効** |
| 価格 | **¥2,500** |

### 3-3. 年額プランを登録する

| 項目 | 入力値 |
|------|--------|
| 製品ID | `premium_annual_25400` |
| 名前 | `プレミアムプラン（年額）` |

**基本プランを追加**

| 項目 | 入力値 |
|------|--------|
| 基本プランID | `annual-base` |
| 有効期間 | **1年** |
| 価格 | **¥25,500** |

### 3-4. 無料トライアルを設定する（任意・推奨）

各基本プランに **「特典を追加」** → **「無料トライアル」**

| 項目 | 推奨設定 |

| 期間 | **7日間** |
| 適用条件 | 新規ユーザーのみ |

## 4. プロモーションコードの扱い

プロモーションコードには **2種類** あります。用途に応じて使い分けてください。

### 種類の比較

| | App Store / Play Store の公式コード | アプリ独自のカスタムコード |
|---|---|---|
| **管理場所** | App Store Connect / Play Console | Firestore（自前） |
| **用途** | メディア・PR・インフルエンサー向け | 友人紹介・SNSキャンペーン・テスター招待 |
| **ストア手数料** | かかる（30%） | **かからない** |
| **コード配布数** | 最大100〜1,000枚（Appleは申請制） | 無制限 |
| **有効期間の柔軟性** | ストア側の制限に準拠 | 自由に設定可 |
| **管理の手間** | ストア管理画面で操作 | Firestoreコンソールで操作 |

---

### 4-1. App Store プロモーションコードの発行

> 審査済みアプリのみ発行可能。アプリ申請と同時には発行できません。

1. App Store Connect → 対象アプリ → **「App内課金」**
2. 対象のサブスクリプション → **「プロモーションコード」**
3. **「コードを作成」** → 枚数を指定（最大100枚/90日、年間1,000枚）
4. CSVまたはリンク形式でダウンロード

**制限事項**
- 1枚のコードで1ユーザーのみ使用可能（ワンタイム）
- 既存サブスクユーザーには適用不可
- 日本のApp Storeのみで有効（国設定に注意）

---

### 4-2. Google Play プロモーションコードの発行

1. Play Console → **「収益化」** → **「プロモーションコード」**
2. **「コードを作成」** → キャンペーン名・コード数・有効期限を設定
3. CSVでダウンロード

**制限事項**
- 1コード1ユーザー
- コードは英数字12〜20文字で自動生成される
- 発行後の取り消しは不可

## 5. Firestore でのカスタムコード管理

アプリ独自のプロモコードシステム（実装済み）の運用方法です。  
ストア手数料なし・自由に設定できるため、**キャンペーンの主力手段** として活用できます。

> **コンソール操作の詳細（手順・フィールド型・トラブルシュート）** は [プロモコード設定ガイド.md](./プロモコード設定ガイド.md) にまとめています。

### 5-1. Firestoreの構造

```
promo_codes/
  └── {コード文字列（大文字英数字）}/
        ├── active: true           // false にすると即時停止
        ├── durationDays: 30       // 何日間有効にするか
        ├── maxUses: 100           // 使用上限（省略で無制限）
        ├── expiresAt: Timestamp   // コード自体の期限（省略で永続）
        ├── usedCount: 0           // 使用済み回数（自動加算）
        └── description: "..."    // 管理用メモ（アプリには非表示）
```

### 5-2. コードの作成手順

1. [Firebase Console](https://console.firebase.google.com) → Firestore
2. `promo_codes` コレクションを開く（なければ作成）
3. **「ドキュメントを追加」** → ドキュメントIDにコード文字列を入力
   - 例：`SUMMER30`、`BETA2025`、`FRIEND50`
4. 以下のフィールドを設定して保存

**設定例**

| 用途 | コード | durationDays | maxUses | expiresAt |
|------|--------|-------------|---------|-----------|
| SNSキャンペーン | `SUMMER30` | 30 | 500 | 2025-08-31 |
| ベータテスター招待 | `BETATESTER` | 90 | 50 | （なし） |
| 友人紹介 | `FRIEND2025` | 14 | （なし） | 2025-12-31 |
| 無制限VIP | `VIP2025` | 365 | 10 | （なし） |

### 5-3. コードの停止・無効化

- **即時停止**：`active` を `false` に変更 → 以降のコード入力が失敗するようになる
- **使用上限に達したもの**：`usedCount >= maxUses` で自動的に弾かれる
- **期限切れコード**：`expiresAt` 以降は自動的に無効

### 5-4. 不正利用への対策（実装済み）

| 対策 | 仕組み |
|------|--------|
| 同一コードの再利用防止 | `usedPromoCodes[]` 配列にユーザーごとに記録 |
| 競合・同時適用防止 | Firestoreトランザクションで排他制御 |
| 既存サブスクへの上乗せ | 期間を積み上げ延長（重複なし） |
| コードの一括停止 | `active: false` で即時全停止 |

### 5-5. 利用状況の確認

Firestore Console で `promo_codes/{コード}` の `usedCount` を見れば、  
何人がそのコードを使ったか即座に確認できます。

より詳細な分析が必要な場合は、Cloud Functions の `redeemPromoCode` ログを  
Firebase Cloud Logging で確認してください。

## まとめ：どちらのコードをいつ使うか

| シーン | 推奨 |
|--------|------|
| App Storeの公式プロモーション・メディア掲載 | App Store プロモコード |
| Google Playの公式プロモーション | Play Console プロモコード |
| SNSキャンペーン・大量配布 | **カスタムコード（Firestore）** |
| ベータテスター・身内への招待 | **カスタムコード（Firestore）** |
| 友人紹介プログラム | **カスタムコード（Firestore）** |
| インフルエンサーへの個別提供 | どちらでも可（ストア手数料を嫌うならカスタム） |
