# プロモコード設定ガイド（Firestore カスタムコード）

このアプリで **ユーザーが入力してプレミアム期間を付与できるプロモコード** は、Firebase の **Firestore** 上の `promo_codes` コレクションで管理します。ストア（Apple / Google）が発行する公式プロモコードとは別物です。

| 種類 | 設定場所 | この資料 |
|------|----------|----------|
| **Firestore カスタムコード**（実装済み） | Firebase Console の Firestore | **本ガイドの主題** |
| App Store の公式コード | App Store Connect | [サブスクリプション設定ガイド §4-1](./サブスクリプション設定ガイド.md#4-1-app-store-プロモーションコードの発行) |
| Google Play の公式コード | Play Console | [サブスクリプション設定ガイド §4-2](./サブスクリプション設定ガイド.md#4-2-google-play-プロモーションコードの発行) |

処理のコードレベルの説明は [サブスクとプロモコード_実装ガイド.md §6](./サブスクとプロモコード_実装ガイド.md#6-カスタムプロモコードの処理) を参照してください。

## 1. 動作の前提

次が揃っていると、カスタムプロモコードが利用できます。

1. **Cloud Functions** がデプロイされ、**`redeemPromoCode`** が利用可能であること（`functions/index.js`）。リージョンは **`asia-northeast1`** です。
2. コードを入力するユーザーが **Firebase Authentication でログイン済み**であること（未ログインでは Callable が拒否されます）。
3. アプリからペイウォール（プロモ入力欄）を開けること。入力後は `SubscriptionService.redeemPromoCode` 経由でサーバーが検証し、**`users/{ユーザーUID}/subscription/status`** を更新します。

コードの大文字・小文字: サーバー側で **trim したうえで大文字に正規化**されるため、ドキュメント ID は **大文字の英数字** にしておくと管理しやすいです（例: `SUMMER30`）。

## 2. Firestore でコードを 1 件追加する手順

1. [Firebase Console](https://console.firebase.google.com) を開き、対象プロジェクトを選択する。
2. 左メニューから **Firestore Database** を開く。
3. **コレクションを開始** または既存ルートから、トップレベルに **`promo_codes`** という名前のコレクションを用意する（既にあればそのまま使う）。
4. **`promo_codes`** を開き、**ドキュメントを追加** を選ぶ。
5. **ドキュメント ID** に、ユーザーへ配布するコード文字列をそのまま入力する（例: `BETA2026`）。  
   - 英字・数字の組み合わせを推奨。記号はサーバー実装や運用ポリシーに合わせて避けると安全です。
6. 下記 [フィールド一覧](#3-フィールド一覧型と意味) に従いフィールドを追加する。  
   **最低限必須**なのは `active`（boolean）と `durationDays`（number）です。
7. **保存** する。

**初回のみ `usedCount` を入れる場合:** `0`（number）を入れておくと、コンソール上で利用回数が追いやすいです。省略した場合も、初回の利用時にサーバー側でインクリメントされます。

**`expiresAt` を使う場合:** フィールドの型は **タイムスタンプ** にする。日付だけ決めたい場合は、その日の 23:59 など JST を意識して UTC で登録するか、期限当日いっぱいまで有効にしたいなら翌日 0:00 直前にするなど、運用で固定します。

---

## 3. フィールド一覧（型と意味）

| フィールド名 | 必須 | 型 | 説明 |
|--------------|------|-----|------|
| `active` | **はい** | boolean | `true` のときのみ利用可能。`false` にすると即座に無効化。 |
| `durationDays` | **はい** | number | 付与するプレミアム期間の **日数**。0 以下は想定外なので避ける。 |
| `maxUses` | いいえ | number | コード全体の **最大利用回数**。未設定（フィールドなし）は **無制限**。 |
| `expiresAt` | いいえ | timestamp | **コード自体**の有効期限。過ぎると新規の入力が拒否される。未設定は「コード期限なし」。 |
| `usedCount` | いいえ | number | **何回使われたか**。通常は `0` で開始し、以降は **サーバーが自動加算**（手で減らすと集計がずれるので注意）。 |
| `description` | いいえ | string | 管理用メモ（キャンペーン名など）。アプリ画面に表示する実装は必須ではない。 |

サーバーは `promo_codes/{ドキュメントID}` をコードとして参照します。**ドキュメント ID = ユーザーが入力するコード**（大文字化後と一致）です。

## 4. 設定パターン例

コピー用の「意味」ではなく、コンソールで追加する値のイメージです。

### 小規模テスト用（限定的）

- ドキュメント ID: `TEST01`
- `active`: `true`
- `durationDays`: `7`
- `maxUses`: `10`
- `description`: `社内テスト用`

### SNS キャンペーン（先着・期限付き）

- ドキュメント ID: `SUMMER2026`
- `active`: `true`
- `durationDays`: `30`
- `maxUses`: `500`
- `expiresAt`: `2026-08-31` 相当のタイムスタンプ（運用で終了日を決める）
- `usedCount`: `0`

### 長期ベータ（人数制限のみ）

- ドキュメント ID: `BETATESTER`
- `active`: `true`
- `durationDays`: `90`
- `maxUses`: `50`
- `description`: `クローズドβ`

### 既にサブスク中のユーザーへの「上乗せ」

同じユーザーがストア課金や別プロモで既に `expiresAt` が未来にある場合、サーバーは **現在の失効日の翌日から** `durationDays` 分を **延長**します（「今日から差し替え」ではなく積み上げ）。未加入なら **今日から** `durationDays` 日が加算されます。

## 5. 停止・修正・監視

| やりたいこと | 操作 |
|--------------|------|
| すぐに止めたい | 該当ドキュメントの `active` を `false` に変更。 |
| 配布数を増やしたい | `maxUses` を大きくする、またはフィールドを削除して無制限にする。 |
| 期限を延ばしたい | `expiresAt` を未来の日時に更新、またはフィールドを削除。 |
| 何回使われたか確認 | `usedCount` を見る（Firestore Console）。 |
| 誰が使ったかの一覧 | 現状の実装では **ユーザーごとの `usedPromoCodes` はユーザードキュメント側**（`users/{uid}/subscription/status`）に入ります。全ユーザーを横断したレポートが必要なら BigQuery エクスポートや Cloud Logging の利用を検討。 |

---

## 6. ユーザー側の操作

1. アプリに **ログイン**する。
2. ペイウォール（例: サイドメニューからプレミアム案内 → シート表示）を開く。
3. プロモコード入力欄にコードを入力し、適用ボタンを押す。
4. 成功すると、付与日数のメッセージが表示され、**プレミアム状態は Firestore の更新に追随**して反映されます。

## 7. トラブルシューティング

| 現象 | 確認すること |
|------|----------------|
| 常に「無効なコード」 | ドキュメント ID が入力内容と一致しているか（大文字小文字はサーバーで大文字化）。`promo_codes` のスペルミス。 |
| 「現在使用できません」 | `active` が `true` か。 |
| 「有効期限が切れています」 | `expiresAt` が過去になっていないか。 |
| 「使用上限に達しました」 | `usedCount` と `maxUses`。必要なら上限を引き上げ。 |
| 「既に使用済みです」 | 同一ユーザーは **同じコードを 2 度使えない** 仕様。別コードを発行する。 |
| そもそも通信エラー | Functions のデプロイ・リージョン、端末のネットワーク、Firebase Auth のログイン状態。 |

Callable の詳細ログは **Google Cloud Console → Logging** で `redeemPromoCode` をフィルタすると追いやすいです。
