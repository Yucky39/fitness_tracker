# Fitness Tracker App

Flutterで作成されたフィットネストラッカーアプリです。  
ログイン後、画面下部のタブで **ホーム（ダッシュボード）**、**食事**、**トレーニング**、**進捗**を切り替えて利用します。画面上部の **プロフィール（アカウント）アイコン**からエンドドロワーを開き、目標・AI・通知・データエクスポートなどをまとめて操作できます。

## セットアップ手順

このプロジェクトは手動でファイルを作成したため、以下の手順で実行環境を整えてください。

1. Flutter SDKがインストールされていることを確認してください。
2. ターミナルでプロジェクトルート（`pubspec.yaml`がある場所）に移動します。
3. 以下のコマンドを実行して、プラットフォーム固有のファイルを生成します。
   ```bash
   flutter create .
   ```
   ※ `android` や `ios` フォルダが生成されます。

4. 依存関係をインストールします。
   ```bash
   flutter pub get
   ```

5. Firebase（認証・Firestore）を利用する場合は、`firebase_options.dart` などプロジェクト用の設定が済んでいることを確認してください。

6. アプリを実行します。
   ```bash
   flutter run
   ```

## アカウントとデータ同期

- **ログイン / 新規登録**: メールアドレスとパスワードで Firebase Authentication を利用します。
- **ログイン時**: Firestore に保存されたデータを端末に取り込み、既存のローカル記録とマージします（同一IDの重複は作りません）。
- **新規登録時**: 登録時点のローカルデータを Firestore にアップロードし、オフラインで溜めた記録を引き継げます。
- **同期対象（実装）**: `food_items`（食事）、`training_logs`（トレーニング）、`body_metrics`（身体測定）。保存・削除時にバックグラウンドで Firestore へ反映する設計です。**食事プリセット・ルーティン・トレーニングプラン等はローカルのみ**のテーブルがあり、Firestore 同期の対象外です。
- **ログアウト**: エンドドロワー下部から実行できます。

## 機能概要

### 0. ホーム（ダッシュボード）

- **カロリー収支**: その日の摂取カロリー、目標、トレーニング由来の推定消費、**歩数由来の推定消費**をまとめて表示し、摂取と運動を差し引いた収支のイメージを確認できます。
- **睡眠**: iOS（HealthKit）/ Android（Health Connect）から **昨夜の睡眠時間** を取得して表示（権限が必要。Web では非対応）。
- **歩数**: 同様にデバイスの歩数と推定消費カロリーを表示。
- **期間サマリ**: 直近 1週間 / 1ヶ月 / 3ヶ月の食事・トレーニング・体重変化などを要約表示。
- **クイックアクセス**: 食事・トレーニング・進捗タブへ遷移。
- **ルーティンリマインダー**: 曜日ごとのトレーニングルーティンが当日に該当する場合に案内（ルーティン本体の編集はトレーニング画面から）。
- **週間カロリーグラフ**: 摂取カロリーの週間推移。

プルダウンで更新すると、睡眠・歩数の同期もまとめて走ります。

### 1. 食事管理

- **日付の切り替え**: 前日・翌日ボタンやカレンダーで、参照する日を変更できます（未来の日は選択不可）。
- **目標設定**: カロリーおよび PFC（タンパク質・脂質・炭水化物）の目標を手入力できます。身長・体重・年齢・性別・活動レベル・目標体重・達成までの週数から、基礎代謝・TDEE に基づき **1日の摂取目標を自動計算** することもできます（**プロフィール**の「カロリー・栄養目標設定」）。
- **食事の追加方法**
  - **手動入力**: 食品名と栄養素を入力。**糖質・食物繊維・ナトリウム** なども入力可能な項目があります。
  - **食品検索**: キーワードで検索。**優先順は (1) 文部科学省 食品成分データベース（日本食品標準成分表・八訂相当の公開API） (2) Open Food Facts (3) アプリ内の目安データ** です。日本語クエリでは成分表が優先されます。
  - **写真で自動分析**: 食事写真から AI が内容を推定（利用には API キー設定が必要。下記「AI 関連」）。
  - **バーコードスキャン**: Open Food Facts から栄養情報を取得（**Web では利用不可**。モバイル向け）。
  - **プリセット**: よく使う食事セットを保存し、ワンタップで追加。**「今日の食事をプリセット保存」**で当日分をまとめて保存可能。
  - **レシピからプリセット**: 食材・分量・調理法を入力し、栄養を集計して名前付きプリセットとして保存（レシピプリセットエディタ）。
- **サマリー・グラフ**: 選択した日の摂取カロリーと PFC の達成状況を表示。過去の推移（トレンド）もグラフで確認できます。
- **栄養アドバイス**: その日の記録に基づき、AI が日本語でアドバイス（API キーとプロバイダ設定が必要）。
- **一覧表示**: グループ表示とタイムライン表示を切り替え可能。

#### AI 関連（プロフィールの「AIキー・トレーニングアドバイス設定」）

- プロバイダ: **Anthropic（Claude）**、**OpenAI**、**Gemini** から選択。**モデル**もドロップダウンで選択可能。
- API キーを端末に保存（セキュアストレージを利用）。
- 栄養アドバイスの口調: **厳しめ / ふつう / やさしめ** から選択。
- **トレーニング AI アドバイス**: 各トレーニング記録で AI 評価を使うかどうかをオンオフ可能（同じ API キー設定を利用）。

### 2. トレーニング記録

- **記録**: 種目名、重量、回数、セット数、インターバル、**RPE（1–10）**、メモを記録・編集・削除できます。
- **インターバルタイマー**: セット間タイマーを画面に表示してカウントダウンできます。
- **履歴**: 過去のトレーニングを一覧表示。当日分のサマリーも表示されます。
- **前回参照**: 同じ種目名の過去記録を入力時に参照しやすくなっています。
- **推定消費カロリー**: 記録内容と体重（進捗で入力した体重など）から推定を表示します。
- **PR 表示**: 種目ごとの自己ベストに該当するかを表示します。
- **推定 1RM**: 記録に基づき種目ごとの推定最大挙上重量を表示するカードがあります。
- **AI プラン作成**（AppBar）: 目標（増量・維持・減量など）、部位・週の日数・強度・器具環境を選び、**設定済みの LLM API** からトレーニングプランを JSON 形式で生成・保存します。
- **ルーティン管理**（AppBar）: 曜日ごとのメニュー名・メモなどを管理。ダッシュボードの当日リマインダーと連動します。
- **ヘルスケア連携**（iOS / Android）: 初回に権限が取れた場合、**過去30日分のワークアウト**を取り込み、トレーニングログに追加できます（重複は除外）。Web では無効です。

### 3. 進捗トラッキング

- **身体測定**: 体重、腹囲、体脂肪率を記録し、グラフで推移を確認できます。
- **集計単位**: 日別・週別・月別でグラフ表示を切り替え可能。
- **画像記録**: 体型の写真を記録に添付できます。
- **画像比較**: 2枚の写真を半透過で重ね、透明度を調整しながら体型の変化を比較。比較対象が複数ある場合は切り替え可能です。
- **BMI など**: 身長・目標体重など（エネルギープロファイル）が入力されていると、表示に反映されます。

### プロフィール（エンドドロワー）

- **カロリー・栄養目標設定**: 手入力および Mifflin–St Jeor ベースの自動計算。
- **AIキー・トレーニングアドバイス設定**: 上記 AI・モデル・栄養の口調・トレーニング評価の有効化。
- **リマインダー通知**: 食事記録・トレーニングのローカル通知時刻（`flutter_local_notifications`）。
- **データ管理**: 食事・トレーニング・身体測定を **CSV** にエクスポートし、共有シートで出力。

## アプリ全体

- **テーマ**: Material 3。ライト / ダークは **システム設定** に追従します。

## 注意事項

- **データの保存**: モバイルでは主に SQLite 相当のローカル DB、Web ではブラウザのローカル保存を用います。ログイン中は上記3テーブル相当のデータが Firestore にも同期される設計です。
- **アプリ削除**: 端末上のデータは失われる可能性があります。アカウントで同期しておくと再インストールや別端末での復元に役立ちます（同期対象外のローカルデータは除く）。
- **AI 機能**: 写真解析・栄養アドバイス・トレーニングプラン生成・トレーニング評価には、各自で API キーを用意しプロフィールで登録する必要があります。
- **バーコード**: Web ビルドではスキャン機能は使えません。
- **Health（睡眠・歩数・ワークアウト）**: **ネイティブ（iOS / Android）のみ**。権限とデータ提供元アプリの設定に依存します。
- **iOS シミュレーター**: カメラ利用時は `Info.plist` にカメラ権限の記述が必要になる場合があります（`flutter create` のデフォルトで不足する場合）。
