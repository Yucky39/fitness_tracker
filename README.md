# Fitness Tracker App

Flutterで作成されたフィットネストラッカーアプリです。  
ログイン後、画面下部のタブで **食事管理**、**トレーニング記録**、**進捗トラッキング**（画像比較付き）を切り替えて利用します。

## セットアップ手順

このプロジェクトは手動でファイルを作成したため、以下の手順で実行環境を整えてください。

1. Flutter SDKがインストールされていることを確認してください。
2. ターミナルでプロジェクトルート（`pubspec.yaml`がある場所）に移動します。
3. 以下のコマンドを実行して、プラットフォーム固有のファイルを生成します。
   ```bash
   flutter create .
   ```
   ※ `android` や `ios` フォルダが生成されます。

4. 依存関係をインストールします。
   ```bash
   flutter pub get
   ```

5. Firebase（認証・Firestore）を利用する場合は、`firebase_options.dart` などプロジェクト用の設定が済んでいることを確認してください。

6. アプリを実行します。
   ```bash
   flutter run
   ```

## アカウントとデータ同期

- **ログイン / 新規登録**: メールアドレスとパスワードで Firebase Authentication を利用します。
- **ログイン時**: Firestore に保存されたデータを端末に取り込み、既存のローカル記録とマージします（同一IDの重複は作りません）。
- **新規登録時**: 登録時点のローカルデータを Firestore にアップロードし、オフラインで溜めた記録を引き継げます。
- **同期対象（例）**: 食事記録、トレーニングログ、身体測定など。保存・削除時にバックグラウンドで Firestore へ反映する設計です。
- **ログアウト**: 画面上部のアカウントメニューから実行できます。

## 機能概要

### 1. 食事管理

- **日付の切り替え**: 前日・翌日ボタンやカレンダーで、参照する日を変更できます（未来の日は選択不可）。
- **目標設定**: カロリーおよび PFC（タンパク質・脂質・炭水化物）の目標を手入力できます。身長・体重・年齢・性別・活動レベル・目標体重・達成までの週数から、基礎代謝・TDEE に基づき **1日の摂取目標を自動計算** することもできます（歯車アイコンの「目標・設定」）。
- **食事の追加方法**
  - **手動入力**: 食品名と栄養素を入力。
  - **写真で自動分析**: 食事写真から AI が内容を推定（利用には API キー設定が必要。下記「AI 関連」）。
  - **バーコードスキャン**: Open Food Facts から栄養情報を取得（**Web では利用不可**。モバイル向け）。
  - **プリセット**: よく使う食事セットを保存し、ワンタップで追加。
- **サマリー・グラフ**: 選択した日の摂取カロリーと PFC の達成状況を表示。過去の推移（トレンド）もグラフで確認できます。
- **栄養アドバイス**: その日の記録に基づき、AI が日本語でアドバイス（API キーとプロバイダ設定が必要）。
- **一覧表示**: グループ表示とタイムライン表示を切り替え可能。

#### AI 関連（目標・設定ダイアログ内）

- プロバイダ: **Anthropic（Claude）**、**OpenAI**、**Gemini** から選択。
- API キーを端末に保存（セキュアストレージを利用）。
- アドバイスの口調: **厳しめ / ふつう / やさしめ** から選択。

### 2. トレーニング記録

- **記録**: 種目名、重量、回数、セット数、インターバル、メモを記録・編集・削除できます。
- **履歴**: 過去のトレーニングを一覧表示。当日分のサマリーも表示されます。
- **前回参照**: 同じ種目名の過去記録を入力時に参照しやすくなっています。
- **推定消費カロリー**: 記録内容と体重（進捗で入力した体重など）から推定を表示します。
- **PR 表示**: 種目ごとの自己ベストに該当するかを表示します。

### 3. 進捗トラッキング

- **身体測定**: 体重、腹囲、体脂肪率を記録し、グラフで推移を確認できます。
- **集計単位**: 日別・週別・月別でグラフ表示を切り替え可能。
- **画像記録**: 体型の写真を記録に添付できます。
- **画像比較**: 2枚の写真を半透過で重ね、透明度を調整しながら体型の変化を比較。比較対象が複数ある場合は切り替え可能です。
- **BMI など**: 身長・目標体重など（エネルギープロファイル）が入力されていると、表示に反映されます。

## アプリ全体

- **テーマ**: Material 3。ライト / ダークは **システム設定** に追従します。

## 注意事項

- **データの保存**: モバイルでは主に SQLite 相当のローカル DB、Web ではブラウザのローカル保存を用います。ログイン中は食事・トレーニング・身体測定などが Firestore にも同期される設計です。
- **アプリ削除**: 端末上のデータは失われる可能性があります。アカウントで同期しておくと再インストールや別端末での復元に役立ちます。
- **AI 機能**: 写真解析・栄養アドバイスには、各自で API キーを用意し「目標・設定」で登録する必要があります。
- **バーコード**: Web ビルドではスキャン機能は使えません。
- **iOS シミュレーター**: カメラ利用時は `Info.plist` にカメラ権限の記述が必要になる場合があります（`flutter create` のデフォルトで不足する場合）。
